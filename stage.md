正式上线前，建议把项目分成 **6 个阶段** 推进。核心路线是：

```text
先稳定本地 AI 闭环
再做 Live2D 人物 + 背景
再做移动端 App
再加入识图
再做产品化测试
最后才做云端 API / 正式部署
```

**阶段 1：本地后端闭环稳定**

目标：先让现在已有系统可靠运行。

重点：

```text
修复中文乱码
稳定 /chat
稳定长期记忆 mem0 + Qdrant
稳定 RAG 检索
稳定 TTS 返回
统一 Live2D 返回字段
Qt 能看到 memory / RAG / Live2D 调试信息
```

完成标准：

```text
文本聊天稳定
长期记忆能写入、检索、参与回复
RAG 能召回正确知识
TTS 不阻塞主流程
/chat 返回结构稳定
```

---

**阶段 2：Live2D 协议与表现系统**

目标：把 Live2D 从“附带字段”升级成正式表现系统。

重点：

```text
定义 live2d 标准协议
emotion → expression 映射
emotion → motion 映射
reply → mouth / audio 映射
scene → background 映射
触摸事件协议
人物状态机
背景状态机
```

V1.0 必须覆盖：

```text
人物模型
表情
动作
口型
眨眼
呼吸
注视
背景
光照
场景切换
```

完成标准：

```text
每次对话都能稳定返回 live2d 指令
Qt 或测试端可以模拟人物动作和背景变化
Live2D 协议不再随便变
```

---

**阶段 3：移动端 App 骨架**

目标：做真正的 App，不是 H5。

推荐方向：

```text
Flutter App + 原生 Live2D SDK Bridge
```

App 首批功能：

```text
登录 / 本地用户身份
Live2D 主舞台
聊天输入
语音播放
聊天历史
记忆状态展示
设置页
调试面板
```

完成标准：

```text
App 能连接本地后端
能显示角色
能聊天
能播放语音
能根据后端 live2d 字段切换动作表情
```

---

**阶段 4：Live2D 人物 + 背景正式落地**

目标：让角色真的“活起来”。

重点：

```text
移动端加载 Live2D 模型
表情切换
动作播放
idle 待机动作
TTS 口型同步
触摸反馈
人物与背景分层渲染
背景随状态变化
性能优化
```

背景系统建议：

```text
room_default
room_night
stage_default
desk_work
sky_evening
memory_soft
```

完成标准：

```text
角色不只是聊天框
回复、表情、动作、语音、背景能联动
移动端帧率稳定
资源加载稳定
切后台/恢复不崩
```

---

**阶段 5：识图功能接入**

目标：让角色“看得见”。

重点：

```text
图片上传
图片理解
OCR
物体识别
场景识别
图片摘要进入对话上下文
vision_graph 接入 main_graph
图片是否写入长期记忆由 graph 判断
```

识图流程：

```text
用户上传图片
→ vision_graph 分析
→ memory_graph 判断是否值得记忆
→ rag_graph 判断是否需要知识
→ llm_graph 回复
→ live2d_graph 输出表情动作背景
```

完成标准：

```text
用户发图后，角色能自然回应
图片内容能参与上下文
重要图片信息可进入长期记忆
Live2D 背景/情绪能随图片场景变化
```

---

**阶段 6：产品化封版**

目标：从“能跑”变成“能上线”。

重点：

```text
错误处理
接口稳定
日志系统
用户数据隔离
隐私策略
记忆删除
RAG 重建工具
App 崩溃恢复
弱网处理
接口超时
TTS 失败降级
模型调用成本控制
```

完成标准：

```text
连续测试稳定
核心功能无阻塞 bug
用户数据可清理
App 安装包可测试
后端 API 文档清楚
关键流程有测试用例
```

---

**阶段 7：云端迁移与正式上线**

你说云端 API 放最后，那就最后做。

迁移内容：

```text
FastAPI 部署到云服务器
Qdrant 迁移云端或云服务器
PostgreSQL 加入用户系统
Redis 加任务队列
对象存储保存音频 / 图片 / Live2D 资源
API Key 全部环境变量管理
HTTPS
域名
日志监控
限流
备份
```

完成标准：

```text
移动端 App 连接云端 API
本地不再承担核心计算
图片、音频、记忆、RAG 都可云端运行
正式用户可使用
```

---

**整体发展方向**

最终产品方向应该收束成一句话：

```text
一个拥有长期记忆、知识库、识图能力、语音能力、Live2D 人物与背景表现的移动端 AI 角色 App。
```

不要把它做成普通 AI 助手。你的差异化是：

```text
AI 智能
长期记忆
角色人格
Live2D 身体
背景场景
多模态理解
陪伴关系
```

上线前最重要的三件事：

```text
1. 她要记得用户
2. 她要像一个稳定角色
3. 她要在手机里“活着”
```

推荐最终路线图：

```text
阶段 1：本地后端闭环稳定
阶段 2：Live2D 协议与表现系统
阶段 3：移动端 App 骨架
阶段 4：Live2D 人物 + 背景落地
阶段 5：识图功能接入
阶段 6：产品化封版
阶段 7：云端迁移与正式上线
```

最关键的一句话：

**先把角色体验做出来，再把基础设施搬上云。**
